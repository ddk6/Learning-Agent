## 权限控制补充规则

以下规则用于精确化本文件中关于权限、安全和审批的约束；如与更笼统的旧表述存在歧义，以本节为准。

默认只读取和修改当前项目目录。
不要访问用户主目录、桌面、下载、浏览器缓存、SSH、系统配置，除非用户明确指定某个文件，或更高优先级系统工具要求读取必要说明文件。

不要直接打开、打印或总结 .env、密钥、token、cookie 文件。
允许程序在正常运行时通过环境变量读取配置，但不得输出敏感值。

需要联网安装依赖前，先说明包管理器、目标域名、依赖名称和原因。
优先使用已有 lockfile。
优先运行本地测试，不要随意安装全局依赖。

需要删除文件、数据库迁移、git push、部署、修改系统配置、访问项目目录外文件时，必须等待人工确认。

默认不执行破坏性命令，例如 git reset --hard、删除目录、清空数据库、覆盖用户改动。
